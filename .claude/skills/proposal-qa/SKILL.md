# proposal-qa — 최종 검토 + 발표평가 Q&A

## 핵심 역할

완성된 HWPX 제출 파일의 최종 품질 점검 + 발표평가 대비 모의 질문 50개+ 생성.

## 단독 사용

```
발표평가 준비해줘
Q&A 생성해줘
최종 점검해줘
제출 전 확인해줘
```

## 파이프라인 사용

`/proposal-build` 다음, 파이프라인의 마지막 단계로 자동 호출.

---

## Step 1: 광탈 패턴 최종 스캔

완성된 HWPX에서 텍스트를 추출하여 광탈 패턴을 최종 점검:

- "세계 최고 수준", "혁신적인", "차세대" 등 추상적 수식어
- "본 사업은 ~을 목적으로 하며" 등 AI 작성 흔적
- "1식" 예산 산출근거
- 정량 목표 중 측정 불가능한 항목
- 출처 없는 시장 데이터

---

## Step 2: 문서 일관성 점검

- 앞표지 과제명 = 본문 과제명 = 요약문 과제명
- 앞표지 연구개발비 = 본문 예산 합계 = 요약문 총 연구개발비
- 앞표지 연구개발기간 = 본문 추진일정 기간
- 성능지표 목표치 = 본문 기재 수치
- 인력 테이블 인원수 = 고용 테이블 상시고용 수

---

## Step 3: 제출 체크리스트

```
[ ] HWPX 파일이 한글에서 정상 열림
[ ] 작성요령 테이블 전부 삭제됨
[ ] 빈 ◦/- 단락 없음
[ ] 커버페이지 플레이스홀더 교체됨
[ ] 이미지가 정상 표시됨
[ ] 앞표지 서명란 확인 (인/직인)
[ ] 요약문 5쪽 이내
[ ] test_hwpx.py 전 항목 PASS
```

---

## Step 4: 발표평가 Q&A 생성

모의 질문 50개+ 생성. 각 질문에:
- 예상 답변 (초안 기반)
- 참조 페이지 / 섹션 번호
- 난이도 (상/중/하)
- 답변 팁

### 질문 카테고리

| 카테고리 | 예시 질문 | 예상 수 |
|---------|---------|--------|
| 기술 실현 가능성 | "TRL 6→8 근거가 뭡니까?" | 10개 |
| 시장/경쟁 | "SOM 산출 근거를 설명하세요" | 8개 |
| 사업화 전략 | "첫 유료 고객은 누구입니까?" | 8개 |
| 예산 적정성 | "이 비목에 1식이 아닌 공수 기반으로 산출한 근거?" | 5개 |
| 팀 역량 | "CTO가 없는데 기술 리스크 관리 방안?" | 5개 |
| 차별점 | "경쟁사 대비 가장 큰 차이 한 문장으로?" | 5개 |
| 리스크 | "핵심 인력 이탈 시 대응 방안?" | 5개 |
| 중간평가 | "6개월 뒤 보여줄 산출물?" | 4개 |

### TIPS 추가 질문 (10개)

- "운영사가 왜 이 팀을 선택했습니까?"
- "선투자금 사용 내역은?"
- "Post-TIPS 후속 투자 계획?"
- "성능지표 목표가 세계최고 대비 왜 이 수준입니까?"
- "글로벌 진출 실현 가능성 근거?"

---

## Step 5: 결과 정리

- `_발표평가_Q&A.md` — 모의 질문 + 답변 + 페이지 참조
- `_최종검토결과.md` — 광탈 패턴 스캔 + 일관성 점검 + 제출 체크리스트
- 사용자에게 최종 보고: 수정 필요 사항 요약
