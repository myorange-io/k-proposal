# proposal-review — 양식 게이트 + 심사위원 채점

## 핵심 역할

완성된 초안을 (1) 양식 충족 여부 사전 검증 + (2) 심사위원 관점 채점 + (3) 킬러 질문 생성 + (4) 약점 보강 지시. 전 항목 60점 이상 + 킬러 질문 답변 불가 0개가 될 때까지 반복.

## 단독 사용

```
이 계획서 검토해줘
심사위원 관점으로 채점해줘
양식 충족 확인해줘
킬러 질문 만들어줘
```

## 파이프라인 사용

`/proposal-write` 다음에 호출. 미달 시 `/proposal-write`에 보강 지시.

---

## 0단계: 양식 충족 게이트 (채점 전 필수)

내용 채점 전 양식 필수 요건을 먼저 검증. **1개라도 미충족 시 채점 불가 → writer에게 보강 지시.**

```
[ ] 앞표지 필수 셀 (과제명, 기관명, 사업자번호, 연구책임자, 연구개발기간, 연구개발비)
[ ] 요약문 필수 셀 (사업명, 과제명, 기간, 총비, 최종목표, 전체내용, 기대효과, 핵심어)
[ ] 사업화 목표 단위 정합 (백만원 vs 천원 — 양식 표기와 값 일치)
[ ] KIPRIS 문장검색 결과 10건 포함 (2-3 섹션)
[ ] 위탁기관명 특정 + 역할분담 근거 (위탁 비중이 있을 때)
[ ] 기술유출 방지대책 기재 (2-3 섹션)
[ ] 고용창출 서술 — 교육프로그램/스톡옵션/내일채움공제/직무보상발명 (3-5 섹션)
[ ] 5-1 안전조치 / 5-2 보안조치 / 5-3 기타 각각 구분 기재
[ ] 커버페이지 플레이스홀더가 실제 값으로 교체됨
[ ] 빈 ◦/- 단락 잔존 없음 (내용 없는 빈 불릿)
```

검증 결과를 `_workspace/03_reviewer/gate_check.md`에 저장.

---

## 1단계: 평가기준별 모의 채점

심사위원 2인(현장 전문가 + 심사 경력 심사역) 시뮬레이션.

| 점수 | 의미 |
|------|------|
| 90-100 | 탁월 — 이 항목만으로 선정 근거 |
| 70-89 | 우수 — 설득력 있으나 일부 보강 |
| 60-69 | 보통 — 통과하지만 경쟁에서 밀림 |
| 40-59 | 미흡 — 탈락 위험 |
| 0-39 | 치명적 — 전체 탈락 결정 |

**공고별 평가기준 자동 매칭:** `준비자료.md`의 평가기준표에서 항목별 채점.

평가기준이 없는 경우 범용 기준:

| 범용 항목 | 배점 | 핵심 질문 |
|----------|------|---------|
| 사업 이해도 | 20 | 문제 정의가 구체적인가? |
| 기술/서비스 경쟁력 | 25 | 차별점이 증명되는가? |
| 사업화 가능성 | 25 | BM이 명확한가? 매출 근거? |
| 수행 역량 | 15 | 팀 경력이 사업과 직결? |
| 기대효과 | 15 | 정량 목표가 측정 가능? |

**60점 미만 항목이 1개라도 있으면 다음 단계 진행 불가.**

### TIPS 전용: 4대 평가항목

| 항목 | 배점 | 핵심 |
|------|------|------|
| 문제인식 | 25 | 시장 데이터, 빈틈 식별, 규제 촉매 |
| 실현가능성 | 25 | TRL, 특허, 모듈별 개발, 성능지표 |
| 성장전략 | 25 | TAM/SAM/SOM, BM, 글로벌, 후속투자 |
| 팀구성 | 25 | 핵심인력 경력, 역할분담, 팀 시너지 |

## 2단계: 킬러 질문 생성

**범용 핵심 질문 (모든 사업 공통):**
1. "이 사업비 규모가 적정한 근거? 절반으로 줄이면?"
2. "경쟁사 대비 핵심 차별점 한 문장으로?"
3. "정량적 목표 달성을 어떻게 증명? 증빙 서류는?"
4. "지원 없이 자체적으로 안 되는 이유?"
5. "6개월 뒤 중간평가 산출물?"

**공고 유형별 추가 질문:**

| 유형 | 추가 질문 |
|------|---------|
| 지역 R&D | "이 지역에서 하는 이유?", "지역 고용 효과?" |
| 예비창업 | "창업 경험 없는데 실행 역량 근거?", "시장 검증?" |
| 창업도약 | "매출 성장 정체 원인?", "스케일업 기여 방법?" |
| TIPS | "운영사가 왜 이 팀을 선택?", "Post-TIPS 투자 시나리오?" |

**답변 가능성 판정:** ✅ 충분 / ⚠️ 약함(보강) / ❌ 불가(필수 추가)

---

## 3단계: 약점 개선

**우선순위:** ❌ 답변 불가 → 60점 미만 → ⚠️ 약함

**규칙:**
- 수치/데이터 추가 시 출처 필수
- 다른 섹션 분량/논리 훼손 금지
- 개선 부분에 `[보강]` 마커
- 회사 자료에 없는 데이터를 지어내지 않음 → 사용자에게 요청

---

## 4단계: 재채점 + 통과 확인

**통과 기준:**
- 전 항목 60점 이상
- 킬러 질문 ❌ 답변 불가 0개
- 광탈 패턴 0개

**통과 실패:** 3단계→4단계 최대 2회 반복. 3회 미달 시 부족 자료 목록 사용자 안내.

**통과 시:** `_workspace/03_reviewer/final_review.md` 생성 → `/proposal-write --humanize`로 진행.
